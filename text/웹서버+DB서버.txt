1. DB 서버
  1) AWS PC(실제 서버)
    (1) 설치
	
    (2) 로컬 접속
	mysql client 프로그램 실행

    (3) 원격 접속 사용자 생성
	CREATE USER '사용자이름'@'%' IDENTIFIED BY '패스워드';

    (4) DB 생성
	CREATE DATABASE 데이터베이스이름;

    (5) 권한 부여
	GRANT ALL PRIVILEGES ON 데이터베이스이름.* TO '사용자이름'@'%';

  2) 본인 PC(개발 환경)
    (1) 백업
	cmd 관리자 권한으로 실행
		cd /
		cd "Program Files"
		cd MySQL
		cd "MySQL Server 8.0"
		cd bin
		mysqldump.exe -u [현재서버의관리자이름] -p [백업할데이터베이스이름] > [저장할파일이름]

		[현재서버의관리자이름] : 현재 접속하려는 서버의 관리자 이름, 보통 root
		[백업할데이터베이스이름] : 내가 파일로 저장하려는 데이터베이스의 이름
		[저장할파일이름] : 저장하고 싶은 파일의 이름

	*액세스가 거부되었습니다. -> cmd를 관리자 권한으로 실행하지 않아서 발생하는 오류
	* -h 옵션이 없으면 명령어를 실행하는 컴퓨터로 접속하는 것


    (2) 복구
	cmd 관리자 권한으로 실행
		cd /
		cd "Program Files"
		cd MySQL
		cd "MySQL Server 8.0"
		cd bin
		mysql.exe -u [서버의사용자이름] -p -h [서버IP] [데이터베이스이름] < [저장된파일이름]

		[서버의사용자이름] : 서버에 생성한 사용자의 이름
		[서버IP] : 접속하고 싶은 서버가 설치된 컴퓨터의 IP 주소
		[데이터베이스이름] : 접속해서 저장된 파일에 백업된 데이터를 복구하고 싶은 싶은 서버에 생성된 DB
		[저장된파일이름] : 백업할 때 저장된 파일의 경로

	*ERROR 1227 (42000) at line ~~~ : Access denied; -> 저장된 파일에서 ~~~번째 줄의 명령어를 실행하다 에러 발생
							저장된 파일에서 해당 명령어를 확인
							중요한 SQL문이 아니면 무시
	
	*ERROR 2003 (HY000): Can't connect to MySQL server on '3.35.11.81' (10060) -> 서버로 접속이 안되는 것
									방화벽 확인
									서버 실행 확인

  3) AWS PC(실제 서버)
    (1) 확인
	mysql client 실행
		SELECT * FROM 복구한DB이름.테이블이름;

	내컴퓨터 우클릭 -> 속성 -> 고급 시스템 설정 -> 고급 탭 -> 환경 변수 -> JAVA_HOME 변수를 추가하고 값으로 openjdk 압축 푼 경로 추가
